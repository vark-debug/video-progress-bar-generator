# UXP UI 设计文档

## 1. UI 设计原则与目标

### 1.1 设计目标

本插件的 UI 设计遵循 Adobe 官方推荐的 UXP 设计规范，以 Spectrum Web Components 为基础组件库，力求与 Premiere Pro 的整体视觉风格保持一致。设计的核心目标是创建一个直观、高效且功能完整的用户界面，使视频创作者能够快速添加专业的进度条效果，而无需离开他们熟悉的编辑环境。界面应在保持简洁的同时，提供足够的灵活性以满足不同创作者的需求，从简单的单章节进度条到复杂的多章节带标记物动画都能轻松实现。

用户在使用本插件时，应该能够在一个统一的界面中完成所有操作：添加和编辑章节、调整样式配置、预览效果、生成进度条，并直接导入到当前序列。避免让用户在多个面板或对话框之间切换，减少操作步骤，提高工作效率。界面的响应性也是重要考量，无论是添加章节、调整滑块还是生成进度条，所有操作都应即时响应，让用户感受到流畅的编辑体验。

### 1.2 设计语言

**视觉一致性**。本插件使用 Adobe Spectrum 设计系统作为视觉语言的基础。Spectrum 是 Adobe 官方推出的设计系统，被所有 Creative Cloud 应用程序采用，因此使用 Spectrum 可以确保插件在视觉上与 Premiere Pro 的其他面板保持一致。界面的配色、字体、间距、阴影等视觉元素都遵循 Spectrum 的规范，包括使用相同的色板（light、dark、darkest）、相同的字体族（Adobe Clean）和相同的组件样式。

**布局节奏**。界面的布局遵循清晰的信息层次结构。最重要的内容——章节列表和预览区域——占据界面的主要空间，次要的配置选项通过可折叠的分组进行组织，辅助信息如状态提示和错误消息则放在底部或模态框中。元素之间的间距使用 8px 作为基准网格，确保视觉上的整齐和一致性。

**交互反馈**。每个可交互的元素都应提供清晰的反馈。按钮在悬停、按下和禁用状态下有不同的视觉表现；输入框在获得焦点时突出显示；进度条在生成过程中动态更新；错误信息用红色高亮显示并附带图标。这种即时反馈帮助用户确认他们的操作已被系统接收，减少误操作的可能性。

## 2. 界面布局结构

### 2.1 整体布局

插件界面采用垂直布局，从上到下依次为：标题栏、主内容区和底部操作栏。主内容区进一步划分为左右两栏：左侧是章节编辑区，右侧是预览和配置区。这种布局平衡了编辑功能和视觉反馈的展示空间，同时保持单屏可读性，避免用户需要滚动页面才能完成常见操作。

### 2.2 响应式设计

UXP 面板的尺寸由用户在 Premiere Pro 中调整，本插件需要适应不同的面板宽度。当面板宽度较宽时（如停靠在右侧面板），采用左右分栏布局，充分利用空间。当面板宽度较窄时（如停靠在浮动窗口），自动切换为单列布局，将预览区域放在章节列表下方。

布局的切换通过 CSS Grid 或 Flexbox 实现，使用媒体查询根据容器宽度选择合适的布局模式。关键的断点设置为：宽度小于 400px 时使用单列布局，宽度大于等于 400px 时使用双列布局。

### 2.3 区域详细说明

**标题栏**包含插件名称和帮助按钮。帮助按钮点击后打开一个简短的使用说明对话框，解释如何添加章节、配置样式和生成进度条。

**章节编辑区**是界面的核心功能区域，提供章节的增删改查操作。每个章节显示时间点、标题和操作按钮。时间点支持直接编辑和拖拽排序。列表底部显示总时长，用户可以直接输入或从 Premiere 序列导入。

**预览区**显示进度条的实时预览效果。预览使用 Canvas 渲染，根据当前的章节和配置实时更新。预览区域下方显示输出信息的摘要，包括分辨率、帧率和时长等。

**样式配置区**是可选的展开区域，默认折叠以保持界面简洁。用户点击标题栏可以展开配置面板。配置项包括背景颜色、文字颜色、分隔线颜色、字体大小、分隔线宽度等，使用颜色选择器和数字输入框实现。

**底部操作栏**包含主要的操作按钮和进度指示器。“生成进度条”按钮是主操作，样式为实心填充。“添加到序列”按钮在生成完成后可用，将进度条视频导入到当前序列。进度条在生成过程中显示操作进度。

## 3. 组件设计

### 3.1 章节列表组件

章节列表是用户管理进度条章节的核心界面。每个章节项是一个可折叠的卡片，包含以下元素：

**时间点编辑器**。时间点显示为可编辑的文本框，格式为 `MM:SS` 或 `HH:MM:SS`，根据总时长自动选择格式。点击文本框进入编辑模式，支持键盘输入和上下箭头调整。输入会自动验证，非数字字符被忽略，超出范围的数值被截断到有效范围。

**标题编辑器**。标题显示为单行文本框，支持最多 200 字符的输入。文本框自动根据宽度截断过长的文本，超出部分用省略号表示。点击编辑时显示完整文本。

**操作按钮组**。每个章节项包含编辑和删除两个操作按钮。编辑按钮打开一个模态框，允许更详细地编辑章节属性，包括多行标题。删除按钮显示确认对话框，防止误操作。

**拖拽手柄**。章节项左侧有一个拖拽手柄，鼠标悬停时显示。用户可以拖拽章节项调整其在列表中的顺序，这个顺序决定了章节在进度条上从左到右的排列。

章节列表使用虚拟滚动技术优化性能。当章节数量较多（如超过 100 个）时，只渲染可见区域的列表项，避免大量 DOM 节点导致的性能问题。

### 3.2 颜色选择器组件

颜色选择器用于选择进度条的背景色、文字色和分隔线颜色。组件提供两种输入方式：直接输入 HEX 颜色值，或通过颜色面板选择。

**输入框模式**。显示一个颜色预览方块和文本输入框。用户可以直接在文本框中输入 HEX 值（如 #FF5733），输入实时验证并更新预览方块的颜色。无前缀的输入（如 FF5733）自动补全前缀。非法格式显示红色边框提示。

**颜色面板模式**。点击预览方块打开颜色面板。面板提供色相条、饱和度/亮度选择器、不透明度滑块和预设颜色网格。用户也可以点击“吸管”按钮从屏幕上取色。颜色面板关闭后自动将选中的颜色填入输入框。

颜色选择器应记住用户最近使用的颜色，存储在配置中作为快捷选项。这减少重复配置相同颜色的操作步骤。

### 3.3 预览画布组件

预览画布是界面上最直观的元素，实时展示当前配置的进度条效果。由于原生模块在后台处理图像渲染，预览功能使用 JavaScript 的 Canvas API 实现，保证即时响应。

**渲染策略**。预览在以下时机更新：章节添加或删除、章节时间或标题修改、样式配置变更、用户手动刷新。每次更新后，使用防抖机制延迟 100 毫秒执行渲染，避免频繁重绘影响性能。对于复杂的渲染逻辑，预览可以降级为较低分辨率，降低计算量。

**渲染内容**。预览渲染与最终输出保持一致的效果，包括背景色、章节分隔线、章节标题和进度条覆盖层。预览中的进度条覆盖层根据当前时间计算，显示默认的 50% 进度，或根据用户拖动的时间轴滑块显示特定进度。

**时间轴滑块**。预览区域下方有一个时间轴滑块，范围从 0 到总时长。用户拖动滑块可以查看不同时间点进度条的效果。滑块上方显示当前时间点的文字标签。

### 3.4 进度指示器组件

进度指示器在生成进度条时显示操作进度。组件由进度条和百分比文本组成，放置在底部操作栏。

**进度条**。进度条是一个水平填充条，从左到右填充表示完成百分比。填充动画使用 CSS transition 实现，使进度变化看起来平滑自然。进度条左侧显示绿色，右侧保持灰色背景。

**百分比文本**。百分比文本显示在进度条旁边，格式为 `60%`。数字使用 tabular-nums 特性确保等宽，避免数字变化时文本跳动。

**状态切换**。进度指示器在不同状态下有不同的表现。空闲状态隐藏进度条。生成状态显示进度条和百分比。完成状态显示绿色勾号和“完成”文字。错误状态显示红色叉号和错误描述。

### 3.5 表单组件

配置表单包含多个输入组件，统一使用 Spectrum 的组件样式。

**数字输入框**。用于字体大小、分隔线宽度等数值输入。组件左侧有标签说明用途，右侧有数字输入框。输入框支持直接输入数字、点击上下微调按钮调整数值、以及拖拽标签调整数值。

**下拉选择框**。用于选择输出格式、帧率预设等选项。下拉框显示当前选中的值，点击展开选项列表。选项列表支持搜索过滤，方便从大量选项中快速定位。

**开关切换**。用于启用或禁用某些可选功能，如添加标记物、自定义字体等。开关切换提供清晰的视觉状态表示。

## 4. 交互流程设计

### 4.1 添加章节流程

添加章节是最常用的操作之一，需要设计得足够便捷。用户可以通过以下三种方式添加章节：

**快速添加**。点击章节列表上方的“添加章节”按钮，在列表末尾添加一个新章节，默认时间为当前列表最后一个章节的时间加 30 秒，标题为“章节 N”。新章节自动进入编辑模式，光标聚焦在时间点输入框上。

**从序列标记添加**。如果用户已经为序列添加了标记，可以点击“从序列标记导入”按钮，将序列中的标记转换为章节。这种方式节省手动输入时间，也确保章节与视频标记一致。

**右键菜单添加**。在章节列表的空白区域右键点击，显示上下文菜单，包含“添加章节”和“在当前位置添加”选项。“在当前位置添加”会在右键点击的位置对应的视频时间点添加章节。

添加章节后，如果章节时间超出当前总时长，自动将总时长调整为章节时间加 30 秒，并提示用户确认。

### 4.2 生成进度条流程

生成进度条是核心功能流程，需要处理好进度反馈和错误处理。

**前置检查**。点击“生成进度条”按钮后，首先进行参数检查：章节列表是否为空、总时长是否大于零、输出路径是否有效。如果检查不通过，显示错误提示并阻止生成。

**进度更新**。确认参数有效后，开始生成过程。界面显示进度指示器，实时更新生成进度。进度信息包括当前阶段（渲染帧、编码视频）和完成百分比。生成过程可以取消，点击“取消”按钮终止操作并清理临时文件。

**完成处理**。生成完成后，进度指示器显示完成状态，“添加到序列”按钮变为可用。如果启用了自动导入设置，进度条视频自动导入到当前序列。显示一个成功通知，包含输出文件路径和“打开文件夹”按钮。

**错误处理**。生成过程中如果发生错误，停止生成并显示错误详情。错误信息使用用户可理解的语言描述问题，并提供可能的解决方案。如果错误是由于文件权限或磁盘空间不足导致，建议用户检查系统设置。

### 4.3 添加到序列流程

将生成的进度条添加到序列需要与 Premiere API 交互，设计应考虑各种边界情况。

**选择轨道**。默认将进度条添加到序列的第一个视频轨道。如果第一个轨道不可用（如被锁定或没有足够空间），自动选择下一个可用的视频轨道。如果所有视频轨道都不可用，提示用户手动调整序列。

**放置位置**。默认将进度条放置在播放头当前位置。用户可以通过界面设置放置在序列开头、播放头位置或序列末尾。放置前检查目标位置是否有足够的空间，如果空间不足给出警告。

**导入完成**。导入成功后，选中新添加的进度条剪辑，通知用户可以使用 Premiere 的效果控件进一步调整。如果需要，可以右键点击进度条剪辑，选择“在新窗口中打开”进行独立编辑。

## 5. 状态管理

### 5.1 应用状态结构

应用状态使用单一数据源模式管理，所有组件从同一个状态对象读取数据。状态结构设计如下：

```javascript
const appState = {
    // 章节数据
    chapters: [
        { id: "uuid-1", time: 0, title: "第一章" },
        { id: "uuid-2", time: 150, title: "第二章" }
    ],
    
    // 配置数据
    config: {
        width: 1920,
        height: 90,
        bgColor: "#FFFFFF",
        textColor: "#111827",
        separatorColor: "#111827",
        fontSize: 24,
        separatorWidth: 8,
        fps: 30,
        totalDuration: 600,
        fontPath: ""
    },
    
    // 合成配置
    compositeConfig: {
        positionY: 1.0,
        scale: 1.0,
        opacity: 1.0
    },
    
    // 序列信息
    sequence: {
        width: 1920,
        height: 1080,
        fps: 30,
        duration: 600
    },
    
    // UI 状态
    ui: {
        isGenerating: false,
        generationProgress: 0,
        generationStage: "idle",
        error: null,
        previewTime: 0,
        configPanelExpanded: false
    },
    
    // 生成结果
    result: {
        imagePath: null,
        videoPath: null
    }
};
```

### 5.2 状态变更通知

状态变更通过发布-订阅模式通知所有订阅者。当状态变化时，调用 setState 方法更新状态，方法内部触发所有订阅者的回调函数。组件在初始化时订阅感兴趣的状态字段，后续这些字段变化时自动重新渲染。

```javascript
class StateManager {
    constructor(initialState) {
        this.state = initialState;
        this.subscribers = new Map();
    }
    
    getState() {
        return this.state;
    }
    
    setState(path, value) {
        // 更新指定路径的值
        const keys = path.split(".");
        let current = this.state;
        for (let i = 0; i < keys.length - 1; i++) {
            current = current[keys[i]];
        }
        current[keys[keys.length - 1]] = value;
        
        // 通知订阅者
        this.notify(path);
    }
    
    subscribe(path, callback) {
        if (!this.subscribers.has(path)) {
            this.subscribers.set(path, new Set());
        }
        this.subscribers.get(path).add(callback);
        
        // 返回取消订阅函数
        return () => {
            this.subscribers.get(path).delete(callback);
        };
    }
    
    notify(path) {
        const callbacks = this.subscribers.get(path);
        if (callbacks) {
            callbacks.forEach(cb => cb(this.state));
        }
    }
}

const stateManager = new StateManager(appState);
```

### 5.3 数据持久化

用户配置需要持久化存储，避免每次打开插件都需要重新配置。UXP 提供 localStorage API 用于存储少量数据。

**自动保存**。配置变更后自动保存到 localStorage，使用防抖机制避免频繁写入。章节列表和样式配置都参与持久化。序列信息不需要持久化，每次打开时从 Premiere 获取最新值。

**加载恢复**。插件启动时从 localStorage 读取保存的配置，与默认值合并后设置到状态管理器。如果 localStorage 中的数据格式与当前版本不兼容，进行数据迁移后再使用。

**数据迁移**。当配置结构发生变化时（如新增配置项），需要处理旧数据的迁移。迁移函数检查版本号，对旧数据进行转换后保存为新格式。

## 6. 无障碍设计

### 6.1 键盘导航

所有功能都应支持键盘操作，确保无法使用鼠标的用户也能正常使用插件。

**Tab 键导航**。Tab 键按照逻辑顺序在可聚焦元素间切换。章节列表项、输入框、按钮等都是可聚焦元素。Tab 顺序遵循从左到右、从上到下的阅读顺序。

**快捷键**。定义以下快捷键提升效率：Ctrl+N 添加新章节，Delete 删除选中章节，Ctrl+S 触发生成，Escape 取消当前操作。快捷键在插件获得焦点时生效。

**焦点管理**。模态对话框打开时焦点移到对话框内，关闭时焦点返回到打开对话框的按钮。错误提示出现时焦点移到错误信息区域。

### 6.2 屏幕阅读器支持

为辅助技术用户提供完整的语义信息。使用 ARIA 标签标记元素的角色、状态和属性。例如，章节列表使用 list 和 listitem 角色，进度条使用 progressbar 角色，颜色选择器使用 slider 角色。

动态内容更新使用 aria-live 区域通知屏幕阅读器。进度变化、错误信息和成功通知都配置为 aria-live="polite"，让用户了解界面变化而不被打断。

### 6.3 视觉可访问性

配色方案需要满足 WCAG 2.1 AA 级别的对比度要求。文字与背景的对比度至少达到 4.5:1，大文本和界面组件的对比度至少达到 3:1。颜色选择器同时提供文字标签和视觉颜色，不依赖颜色单独传递信息。

焦点状态使用明显的视觉指示，默认的浏览器焦点环可能不够清晰，使用自定义的焦点环样式提高可见性。禁用状态的元素降低透明度，同时保持可读性。

## 7. 错误处理界面

### 7.1 错误提示设计

错误信息需要清晰地传达问题所在和解决方法。使用红色图标和文字标识错误，避免仅依赖颜色传递信息。

**内联错误**。表单输入验证错误显示在输入框下方，用红色文字描述错误原因。例如，时间输入框在输入非法值后显示“请输入有效的时间格式”。

**错误摘要**。表单提交失败时，在表单顶部显示错误摘要，列出所有需要修正的问题。点击错误摘要中的链接跳转到对应的输入框。

**通知错误**。操作过程中发生的错误使用通知横幅显示，固定在界面上方，可手动关闭。通知包含错误描述、可能的解决方案和建议的操作。

### 7.2 错误恢复策略

不同类型的错误有不同的恢复策略。

**输入错误**。用户修正输入后自动清除错误状态，不需要额外操作。

**资源错误**。文件不存在或无权限访问时，提示用户检查文件路径或权限设置，并提供“浏览”按钮打开文件选择器。

**系统错误**。内存不足或磁盘空间不足时，提示用户关闭其他应用程序或清理磁盘空间。显示当前资源使用情况的估计值。

**Premiere 错误**。与 Premiere API 交互失败时，提示用户检查序列是否已打开，或尝试重启 Premiere。
