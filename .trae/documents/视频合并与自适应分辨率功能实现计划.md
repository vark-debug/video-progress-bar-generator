# 视频合并与自适应分辨率功能实现计划

## 1. 需求概述

### 1.1 用户决策（已确认）

| 决策项 | 选择 |
|--------|------|
| 进度条位置 | 覆盖在视频底部（总高度不变） |
| 宽度处理 | 保持原尺寸，居中显示 |
| 分辨率自适应 | 1920 宽 → 90 高（1080P），其余按比例计算 |
| 无视频时 | 只生成进度条（保持现有逻辑） |

### 1.2 高度比例公式

```
进度条高度 = 视频宽度 / 1920 * 90
```

| 视频宽度 | 进度条高度 |
|----------|------------|
| 3840 (4K) | 180 |
| 2560 (2K) | 120 |
| 1920 (1080P) | 90 |
| 1280 (720P) | 60 |
| 854 (480P) | 40 |

## 3. 改造方案

### 3.1 新流程：合并用户视频 + 进度条

```
┌─────────────────────────────────────────────────────────────────┐
│  1. 用户导入视频 → Video.js 加载                                  │
│  2. 用户点击"生成视频"                                            │
│  3. 前端发送：进度条配置 + 视频文件路径                             │
│  4. 后端：                                                        │
│     a. 读取用户视频分辨率                                          │
│     b. 计算自适应进度条高度                                        │
│     c. 生成进度条 PNG（宽度=视频宽度，高度按比例）                   │
│     d. FFmpeg 合并：用户视频 + 进度条 PNG                          │
│  5. 返回合并后的视频                                              │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 FFmpeg 合并策略

当有用户视频时：
1. 输入 0：用户视频文件
2. 输入 1：进度条 PNG（透明背景）
3. 滤镜链：
   - 用户视频缩放到指定尺寸（如果需要）
   - 进度条叠加在底部中央

```bash
# 新 FFmpeg 命令结构
ffmpeg -y -i user_video.mp4 -i progress_bar.png
  -filter_complex "
    [0:v]scale=width:height,format=yuv420p[video];
    [1:v]scale=width:height[bar];
    [video][bar]overlay=x=(W-w)/2:y=(H-h)[out]
  "
  -map [out] output.mp4
```

```python
def get_video_resolution(self, video_path: str) -> tuple[int, int]:
    """使用 ffprobe 获取视频分辨率"""
    import subprocess
    ffmpeg = self.find_ffmpeg()
    result = subprocess.run(
        [
            ffmpeg.replace("ffmpeg", "ffprobe"),
            "-v", "error",
            "-select_streams", "v:0",
            "-show_entries", "stream=width,height",
            "-of", "csv=s=x:p=0",
            str(video_path)
        ],
        capture_output=True,
        text=True
    )
    # 解析结果如 "1920x1080"
    if result.stdout:
        w, h = result.stdout.strip().split("x")
        return int(w), int(h)
    return 1920, 1080  # 默认值
```

```python
def video_info():
    if "video" not in request.files:
        return jsonify({"success": False, "message": "请上传视频文件"}), 400

    video_file = request.files["video"]

    # 保存到临时文件
    import tempfile
    with tempfile.NamedTemporaryFile(delete=False, suffix=".mp4") as tmp:
        video_file.save(tmp.name)

    try:
        creator = ProgressBarCreator()
        width, height = creator.get_video_resolution(tmp.name)
        # 计算推荐的进度条高度
        progress_height = round(width / 1920 * 90)
        return jsonify({
            "success": True,
            "width": width,
            "height": height,
            "progress_height": progress_height
        })
    finally:
        os.unlink(tmp.name)
```

```javascript
async function generateVideo() {
    if (!chapters.length) {
        setStatus("请先添加章节", "error");
        return;
    }

    const startedAt = Date.now();
    setBusy(true, `准备生成视频：0.00%│用时 ${formatElapsed(0)}`);
    $("generateVideoBtn").disabled = true;
    generatedVideo = false;

    try {
        const payload = requestPayload();

        // 如果有视频，附加视频文件
        if (currentVideoFile) {
            const formData = new FormData();
            formData.append("video", currentVideoFile);
            formData.append("config", JSON.stringify(payload));

            // 使用 FormData 发送请求
            const result = await fetch("/generate_video_with_video", {
                method: "POST",
                body: formData
            }).then(r => r.json());

            if (!result.success) throw new Error(result.message || "视频生成失败");
            await watchVideoProgress(result.job_id, startedAt);
        } else {
            // 无视频时使用原有逻辑
            const result = await postJson("/generate_video", requestPayload());
            if (!result.success) throw new Error(result.message || "视频生成失败");
            await watchVideoProgress(result.job_id, startedAt);
        }
    } catch (error) {
        setStatus(error.message, "error");
    } finally {
        $("generateVideoBtn").disabled = false;
        setBusy(false);
    }
}
```

```python
@app.route("/generate_video_with_video", methods=["POST"])
def generate_video_with_video():
    if "video" not in request.files:
        return jsonify({"success": False, "message": "请上传视频文件"}), 400

    if "config" not in request.form:
        return jsonify({"success": False, "message": "缺少配置"}), 400

    try:
        config = json.loads(request.form["config"])
        chapters = config.get("chapters", [])
        if not chapters:
            return jsonify({"success": False, "message": "请先添加章节"}), 400

        # 保存用户视频到临时文件
        video_file = request.files["video"]
        import tempfile
        with tempfile.NamedTemporaryFile(delete=False, suffix=".mp4") as tmp:
            video_file.save(tmp.name)
            user_video_path = tmp.name

        creator = ProgressBarCreator()

        # 获取视频分辨率
        video_width, video_height = creator.get_video_resolution(user_video_path)
        progress_height = round(video_width / 1920 * 90)

        # 更新配置
        config["width"] = video_width
        config["height"] = progress_height

        job_id = uuid.uuid4().hex
        run_video_job(job_id, config, chapters, user_video_path)

        return jsonify({"success": True, "job_id": job_id})

    except Exception as e:
        return jsonify({"success": False, "message": str(e)}), 500
```

```python
def run_video_job(job_id: str, config: dict, chapters: list, user_video_path: str = None):
    jobs[job_id] = {"status": "running", "progress": 0, "message": ""}

    try:
        creator = ProgressBarCreator()
        output_path = creator.create_video(config, chapters, progress_callback, user_video_path)
        jobs[job_id]["status"] = "done"
        jobs[job_id]["download_url"] = f"/download_video?path={output_path}"
    except Exception as e:
        jobs[job_id]["status"] = "error"
        jobs[job_id]["message"] = str(e)
```

## 6. 兼容性与边界情况

### 6.1 无视频时
- 保持现有的纯进度条生成逻辑
- 不发送视频文件，使用原有 `/generate_video` 接口

### 6.2 分辨率适配
- 进度条高度 = `视频宽度 / 1920 * 90`
- 进度条宽度 = 视频宽度（保持一致）
- 进度条居中显示在视频底部

### 6.3 视频格式支持
- 后端使用 ffprobe 获取分辨率，支持所有 FFmpeg 支持的格式
- 前端 Video.js 已支持的格式均可

### 6.4 错误处理
- 视频文件损坏 → ffprobe 失败 → 返回错误信息
- 视频分辨率获取失败 → 使用默认值 1920x1080
